# Процедура оновлення додатку та ML-моделі

Оновлення проєкту зазвичай стосується двох аспектів: оновлення логіки інтерфейсу (`main.py`) або оновлення ваг нейромережі (`best.pt`) на більш точну версію.

## 1. Підготовка до оновлення
1. **Планування часу простою:** Оскільки це Desktop-додаток, оновлення вимагає повного закриття програми користувачем. Простій складе близько 2-3 хвилин.
2. **Перевірка сумісності:** Переконайтеся, що нова модель `best.pt` натренована саме на архітектурі YOLOv11, а не попередніх версіях, інакше бібліотека `ultralytics` видасть помилку.
3. **Створення резервних копій (Backup):**
   * Скопіюйте поточний файл `best.pt` і перейменуйте його на `best_backup.pt`.
   * Зробіть копію поточного файлу `main.py` -> `main_backup.py`.

## 2. Процес оновлення
1. **Зупинка служб:** Закрийте графічне вікно Tkinter. Переконайтеся через Диспетчер завдань, що процес `python.exe` не висить у фоні.
2. **Розгортання нового коду/моделі:**
   * Замініть старий файл `best.pt` на новий.
   * Якщо оновлюється логіка, замініть `main.py`.
3. **Міграція даних:** Не вимагається.
4. **Оновлення конфігурацій:** Перевірте `requirements.txt`. Якщо додалися нові бібліотеки, активуйте середовище та запустіть `pip install -r requirements.txt`.

## 3. Процедура відкату (Rollback) - у разі невдалого оновлення
Якщо після оновлення додаток не запускається або модель видає хибні результати:
1. Закрийте додаток.
2. Видаліть новий (проблемний) файл `best.pt`.
3. Перейменуйте резервну копію `best_backup.pt` назад на `best.pt`.
4. Видаліть проблемний `main.py` та поверніть `main_backup.py`.
5. Запустіть додаток - система повернеться до останнього стабільного стану.
