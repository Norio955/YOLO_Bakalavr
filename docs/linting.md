# Code Quality & Linting Documentation

## 1. Overview / Огляд
Для забезпечення якості коду, відповідності стандартам PEP 8 та уникнення помилок у проєкті використовуються наступні інструменти:

* **Pylint**: Основний лінтер для статичного аналізу коду Python. Перевіряє наявність помилок, "запахів коду" та відповідність стандартам кодування.
* **Black**: Автоматичний форматувальник коду. Забезпечує єдиний стиль оформлення без необхідності ручних правок.
* **Mypy**: Статичний аналізатор типів. Дозволяє знайти помилки, пов'язані з типами даних, ще до запуску програми.

---

## 2. Configuration / Налаштування
Конфігурація інструментів знаходиться у кореневих файлах проєкту:

### Pylint Rules (.pylintrc)
* **Ignored Errors**:
    * `no-member`: Вимкнено для бібліотеки OpenCV (`cv2`), оскільки Pylint некоректно розпізнає динамічні модулі C++.
    * `import-error`: Вимкнено для середовища pre-commit (ізольоване оточення).
* **Disabled Warnings**: `missing-module-docstring`, `missing-class-docstring`, `missing-function-docstring` (для спрощення розробки GUI), `too-many-instance-attributes`, `broad-exception-caught`.
* **Format**: Максимальна довжина рядка збільшена до **120** символів.

### Mypy (.pre-commit-config.yaml / pyproject.toml)
* `ignore_missing_imports`: Увімкнено для сумісності з бібліотеками без type hints (наприклад, старі версії `cv2` або `ultralytics`).

## 3. Automation (Git Hooks) / Автоматизація
У проєкті налаштовано **Pre-commit hooks**. Це гарантує, що "брудний" код не потрапить у репозиторій.

**Як це працює:**
Перед кожним комітом (`git commit`) автоматично запускаються:
1.  **Black** — форматує код. Якщо файл змінено, коміт скасовується, і потрібно додати файл знову.
2.  **Pylint** — перевіряє якість. Якщо рейтинг нижче порогового, коміт блокується.
3.  **Mypy** — перевіряє типи.

---

## 4. Integration & Usage / Інструкція з запуску

### Комплексна перевірка (Script)
Для запуску всіх перевірок однією командою створено скрипт `run_lint.bat`.
Запуск у терміналі:
```powershell
./run_lint.bat
```

### Ручна перевірка
 Якщо потрібно запустити інструменти окремо:

Форматування:
```black .```

Аналіз якості:
```pylint main.py```

Перевірка типів:
```mypy main.py```
