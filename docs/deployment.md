# Інструкція з розгортання (Production Environment)

Цей документ призначений для інженерів з розгортання, які встановлюють додаток на комп'ютери кінцевих користувачів.

## 1. Вимоги до апаратного забезпечення
Додаток обробляє відео в реальному часі, тому має специфічні вимоги до "заліза":
* **CPU:** Intel Core i5 / AMD Ryzen 5 (або краще).
* **RAM:** Мінімум 8 GB (рекомендовано 16 GB).
* **GPU (Опціонально, але рекомендовано):** NVIDIA GPU з підтримкою CUDA для досягнення 30+ FPS.
* **Диск:** Близько 1 GB вільного простору (Python + бібліотеки + ваги моделі).

## 2. Необхідне програмне забезпечення
* Python 3.9+ (Windows Installer).
* Microsoft Visual C++ Redistributable (необхідно для коректної роботи OpenCV на Windows).

## 3. Налаштування мережі та серверів
* **Мережа:** Додаток працює повністю автономно (Offline). Доступ до інтернету не потрібен (всі ваги моделі зберігаються локально).
* **Сервери та СУБД:** Не використовуються. Збереження артефактів відбувається у локальну файлову систему (папки `saved_results` та `logs`).

## 4. Розгортання коду (Кроки для Windows)
1. Скопіювати архів з релізною версією проєкту на цільовий ПК.
2. Розархівувати у зручну директорію (наприклад, `C:\RoadSignDetector`).
3. Створити віртуальне середовище та встановити залежності (`pip install -r requirements.txt`).
4. Налаштувати змінну оточення у файлі `.env`: `APP_LOG_LEVEL=INFO` (щоб не засмічувати диск дебаг-логами).
5. Створити ярлик для файлу `start_prod.bat` на робочому столі користувача.

## 5. Перевірка працездатності (Smoke Test)
Щоб переконатися, що розгортання успішне:
1. Запустіть додаток через `start_prod.bat`.
2. Натисніть "Upload Photo" та оберіть тестове зображення зі знаком.
3. Натисніть "Process Photo". Якщо на екрані з'явився знак із рамкою і програма не "вилетіла" — система готова до експлуатації.
